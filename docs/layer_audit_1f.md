# 1F レイヤー分類監査 (`dxf_output/1階床スリーブ図.dxf`)

**実施日**: 2026-05-20  
**目的**: 現分類規則 (`sleeve_checker/layer_classifier.py`) が業界規約と実データに対して妥当か検証。

## 1. レイヤー命名規則の出自 (リサーチ結果)

| 検証対象 | 該当 |
|---|---|
| JACIC CAD製図基準 (H13/H14/H17) | **無し** |
| 国交省 CAD製図基準 (R2) | **無し** |
| SXF Ver.3.1 仕様 | **無し** |
| 福井県営繕レイヤ基準 | **無し** |

→ **`A211 / A858 / C131 / C151 / F102 / F104 / F105 / F107 / F108_X / F121 / F155 / F201 / F202 / F203 / F305 / F306 / F308` は JACIC/官庁公式仕様ではない**

実態は **竹中工務店社内規約 + Tfas/Rebro 系設備CAD慣行** の混在 (公開ドキュメント未公開、設計BIMツールマスタの一部と推測)。プレフィクス `[空調]/[衛生]/[電気]/[建築]/[基本]` は CADWe'll Tfas のカテゴリタグ。

**現コードベース** (`layer_classifier.py` の「竹中工務店の標準レイヤー命名規則」というコメント) は **正確**。JACIC/SXF とは別物と認識して運用すべき。

## 4. 主な改善提案 (優先順)

### 優先度高 (実害あり)

1. **`エレベーター_間柱` → 柱・仕上線へ変更**  
   現在「乾式壁」になっているのは間違い。実装変更点: `layer_classifier.py:118` の規則順で `エレベーター|ＥＶ|EV` を `A444|間柱` より **前** に移動。

2. **`[電気]通り芯（壁芯）..` → 通り芯へ**  
   `壁芯` キーワード優先で「不要」に落ちている。`(re.compile(r"通り芯"), "通り芯")` をより前に移すか、`C151|壁心|壁芯` のパターンを `(?:^|[^（])C151|(?:^|[^（])壁心|(?:^|[^（])壁芯` のように「カッコ内に含まれる」場合を除外。

### 優先度中 (整合性問題)

3. **`鉄骨対応_三和タジマ` を「不要」へ**  
   メーカー打込金物として `打込金物_三和タジマ` と同等扱い。

### 優先度低 (未確認だが妥当に動いてる)

4. F4XX 系の細部 (`F406 CB` / `F407 ALC` / `F411 電気打込BOX` / `F412 衛生打込BOX` / `F413 空調躯体貫通孔` / `F414 空調打込BOX`) — これらは 1F DXF には実エンティティ無いが、規則上は捕捉されてない。プロジェクト次第で要追加。

## 5. オンライン検証できなかった項目

公開資料に対応表が無いため未検証:

1. `A8XX = 設備記号系` の正式定義 (A858 のレイヤー名称根拠)
2. `F108_X` の竹中公式サブ番号一覧 (`F108_1`, `F108_6` の有無)
3. `F155 スラブレベルハッチング` が竹中公式に「FL値の色分け面」として定義されているか
4. `A858` 以外にスリーブを描画する派生レイヤ (例: 設備D/M系) の有無

竹中工務店または NYK Systems (Rebro 開発元) / ダイテック (Tfas 開発元) に直接照会が必要。

## 6. 結論

- 現分類 (`layer_classifier.py`) は **大筋で妥当**。主要要素 (スリーブ・スラブ・柱・梁・壁・通り芯・FL) の規則は全て命名規則と整合。
- **重大な誤分類は 2 件** (エレベーター_間柱、[電気]通り芯（壁芯）..)。修正は規則順の入れ替えで対応可能。
- **規約は JACIC/SXF 公式ではなく竹中社内 + Tfas/Rebro 慣行**。`layer_classifier.py` のコメント表現は正確。
- **未公開部分の検証** は竹中担当者への照会が必要。
